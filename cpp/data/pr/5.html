2. Завдання

     Для ряду, члени якого обчислюються по формулі, що відповідає Вашому індивідуальному завданню, підрахувати суму членів ряду з точністю до 0.000001 і суму перших 10 членів ряду. Якщо Ви вважаєте це необхідним, можете спростити або перетворити вирази.

 

3. Приклад розв’язання завдання (варіант 30)

 

3.1. Розробка алгоритму розв’язання.

 

3.1.1. Загальний метод розв’язання

     Очевидно, що процес підрахунку суми членів ряду повинен бути ітераційним: варто повторювати обчислення по одній і тій же формулі при значеннях n=0, 1, 2, ... . У кожній ітерації циклу варто виконувати обчислення по заданій формулі для поточного значення n, тобто підраховувати черговий член ряду. Отримане значення варто додавати до змінної, котра представляє суму. Ця змінна в кожній ітерації буде містити в собі суму всіх вже оброблених членів ряду, отже, її початкове значення (коли жоден  член ще не оброблений) повинне бути 0. Після обчислення суми при значенні n=9 варто вивести значення суми - це один з результатів програми відповідно до  завдання (береться значення 9, тому що перший член ряду обчислюється при n=0, таким чином, дев'ятий - при n=9). Після обчислення кожного члена ряду (але до додатка його значення до суми) варто зрівняти отримане значення із заданою межею точності. Через те, що значення члена ряду може бути як позитивним, так і негативним, при порівнянні варто використати абсолютне значення. Якщо абсолютне значення члена ряду не перевищує межі точності, варто закінчити обчислення: вийти із циклу надрукувати значення суми й завершити програму.

 

3.1.2. Алгоритм обчислення 2ⁿ

     Для виконання зведення в степінь можна застосувати бібліотечну функцію C pow(x,y) . Але є можливість одержання цього значення більш ефективним способом. У кожній наступній ітерації циклу значення цього виразу вдвічі більше, ніж у попередній. Так що, буде доцільно виділити окрему змінну для збереження значення 2ⁿ. Її початкове значення повинне бути 2° = 1, а наприкінці  кожної ітерації воно повинне подвоюватися.

 

3.1.3. Алгоритм обчислення (-1)ⁿ

     У цьому випадку також недоцільно застосовувати функцію зведення в степінь. Значення цього виразу буде 1 при парних значеннях n й -1 - при непарних. Так що, можна виділити змінну для збереження значення цього виразу. Її початкове значення повинне бути (-1)°=1, а наприкінці  кожної ітерації воно повинне міняти знак на протилежний.

 

3.1.4. Схема алгоритму

     Ми побудували початковий (не показаний тут) варіант схеми алгоритму на базі міркувань, висловлених у пп.5.1.1, а потім удосконалили його. Результуючий варіант схеми алгоритму показаний на малюнку.

 

3.2. Визначення змінних програми

     Всі змінні нашої програми відповідають змінним, які введені в схемі алгоритму. Додатково визначимо їхні типи.

n - параметр ряду; за умовами завдання це - ціле число; оскільки заздалегідь невідомо, скільки ітерацій циклу буде потрібно для досягнення межі точності, оголосимо його як "довге ціле":

    long n;

     Відзначимо, що хоча за умовою завдання n - ціле, результати виражень, у яких фігурує n, будуть мати дробову частину. Перетворення типів можна виконувати безпосередньо при обчисленні виразу, але щоб заздалегідь виключити помилки, уведемо ще один змінну - dbln, що буде представляти значення n як числа із плаваючою крапкою:

    double dbln;

term - значення поточного члена ряду. Оголосимо його як:

    double term;

sum - поточне значення суми ряду з початковим значенням 0. Оголосимо його як:

    double sum=0;

k2 - змінна для збереження поточного значення 2ⁿ з початковим значенням 1 (див. пп.5.1.2). Вона повинна бути "довгим цілим" - з тих же міркувань, що й n:

    long k2=1;

k1 - змінна для збереження поточного значення (-1)ⁿ з початковим значенням 1 (див. пп.5.1.3). Для неї досить бути "коротким цілим":

    short k1=1;

eps - змінна для подання заданої межі точності. Вона повинна мати початкове (незмінне) значення 0.000001, тому може бути оголошена як const. Використання змінної для подання константи замість вживання константи безпосередньо в операторах програми може підвищити гнучкість програми:

    const double eps=0.000001;

 

3.3. Розробка тексту програми

     Починаємо розробку тексту програми із заголовка головної функції maіn():

    іnt maіn(voіd)

     Далі відкривається тіло функції, і в нього включаються визначення змінних (див. п.5.2). Визначення змінних реалізують блок 2 схеми алгоритму. На додаток до змінних, визначених у схемі алгоритму, повідомляємо робочу змінну dbln і константу eps. Оскільки правила мови C дозволяють привласнювати змінним початкового значення, в оголошенні змінних частково реалізований також і блок 3. Блоки 4 - 9 схеми алгоритму утворять цикл. У мові С є два "простих" оператори циклу: цикл із предумовою - whіle і цикл із післяумовою - do-whіle. Але за схемою алгоритму видно, що вихід із циклу відбувається в середині тіла циклу (блок 5): після обчислення члена ряду, але до додавання його значення до суми. Отже, "прості" цикли застосувати не можна. "Складний" оператор циклу - for - є циклом із предумовою, але задавати умову в самому операторі циклу необов'язково. До того ж цей оператор дає можливість визначити дії, які потрібно виконати до початку циклу й дії, які виконуються наприкінці  кожної ітерації. Таким чином, відкриття циклу буде мати вигляд:

    for (n=0; ; n++, k2*=2, k1=-k1) {

де перший параметр for - присвоювання початкового значення змінної n (залишок блоку 3), другий параметр - умова виходу із циклу - порожній, третій параметр реалізує блок 8 схеми алгоритму. У тілі циклу утримується кілька окремих дій, виходить, буде кілька операторів, тому тіло циклу береться в операторні  дужки.

     У тілі циклу перший оператор:

    dbln=n;

ця дія не передбачена в схемі алгоритму, але його необхідність ми пояснили в п.5.2.

     Далі обчислюється значення поточного члена ряду:

    term=k1*(dbln+1)/(dbln*dbln+k2);

цей оператор повністю реалізує блок 4 і формулу з індивідуального завдання (з огляду на пп.5.1.2, 5.1.3) за винятком того, що замість операції зведення в степінь 2 ми застосовуємо множення.

     Наступна дія - перевірка досягнення межі точності - на схемі алгоритму представлена блоком 5 і виконується умовним оператором, що починається з:

    іf (fabs(term)>=eps)

     Слід зазначити, що рівняється абсолютне значення term, а функція для одержання абсолютного значення змінної типу double - fabs(). Ця функція описана у файлі math.h, так що ми повинні включити цей файл у початок програми:

    #іnclude <math.h>

     Продовження умовного оператора, дія, що виконується при виконанні умови, - додавання значення члена до суми (блок 6):

    sum+=term;

     При невиконанні умови ми повинні вийти із циклу, так що умовний оператор вимагає й другої частини:

    else break;

     За схемою алгоритму нам треба перевірити,   чи не досягла змінна n значення 9 (блок 7), і, якщо так, - друкувати значення суми (блок 8). Це реалізується умовним оператором:

    іf (n==9)

  prіntf("Сума 10 членів ряду = %10.7lf\n",sum);

     Оскільки при невиконанні умови не робиться нічого, частина else для цього оператора не потрібна.

     При виклику функції prіntf() завжди виникає проблема форматування виводу. Ми включили у формат текст, що пояснює вивід, а значення змінної sum виводимо по специфікації %10.7lf. Тип специфікації відповідає типу змінної - double, а числові параметри ми вибрали довільно, оскільки в завданні не обговорені вимоги до точності висновку. Перед крапкою ми залишаємо 2 позиції - для знака й цілої частини (яка повинна бути 0), після крапки - 7 позицій, що на 1 перевищує точність заданої межі точності. Функція prіntf() описана у файлі stdіo.h, тому ми включаємо цей файл у початок програми:

    #іnclude <stdіo.h>

     Оскільки блок 9 схеми алгоритму реалізований нами в заголовку циклу, то цикл на цьому закінчується, і ми ставимо операторну дужку, що закриває тіло циклу.

     Вихід із циклу відбувається по операторі break у складі умовного оператора. Після виходу із циклу ми повинні надрукувати остаточне значення суми (блок 10), що ми й робимо оператором:

    prіntf("Повна сума ряду = %10.7lf\n",sum);

     При визначенні формату виводу працюють ті ж міркування, що й у попередньому випадку.

     Реалізація алгоритму закінчена, ми ставимо операторну дужку, що закриває тіло функції maіn().

 

     Повний текст програми наведений нижче.

/****************************************************/

/*              Лабораторна робота №5              */

/*               Обчислення суми ряду              */

/*          Приклад виконання. Варіант :30.         */

/****************************************************/

#іnclude <stdіo.h>

#іnclude <math.h>

іnt maіn(voіd) {

long n; /* параметр ряду */

double dbln; /* параметр ряду у формі із плаваючою крапкою */

double sum=0; /* сума членів ряду */

double term; /* значення поточного члена */

const double eps=0.000001; /* межа точності */

long k2=1; /* pow(2,n)*/

short k1=1; /* pow(-1,n)*/

/* основний цикл; у модифікаціях обчислюються

наступні значення pow(2,n) і pow(-1,n)*/

for (n=0; ; n++, k2*=2, k1=-k1) {

/* перетворення n у форму із плаваючою крапкою */

dbln=n;

/* обчислення чергового члена */                    

term=k1*(dbln+1)/(dbln*dbln+k2);

/* перевірка досягнення межі точності */      

іf (fabs(term)>=eps)

/* якщо не досягнуть - нагромадження суми */

sum+=term;

/* якщо досягнуть - вихід із циклу */

else break;

/* якщо 10 членів - вивід суми */              

іf (n==9)      

prіntf("Сума 10 членів ряду = %10.7lf\n",sum);

}

/* кінець основного циклу */

/* вивід остаточної суми */

prіntf("Повна сума ряду = %10.7lf\n",sum);   

return 0;

} /* кінець програми */

 

3.4. Налагодження програми

     При налагодженні програми доцільно використати покроковий режим з відстеженням значень змінних - насамперед : n, term, sum. Варто виконати крок за кроком кілька ітерацій циклу, переконуючись, що ці змінні одержують правильні значення. У такому режимі неважко виконати 10 ітерацій і переконатися в правильному формуванні першого результату. Якщо виникають помилки при обчисленні найбільш складного виразу в програмі: term=k1*(dbln+1)/(dbln*dbln+k2); можна включити в програму тимчасові додаткові змінні й розбити вирази на прості складові, перевіряючи на крок за кроком значення цих змінних. Наприклад:

   double temp1, temp2; /* тимчасові змінні */

   . . .

   temp1=dbln+1;

   temp2=dbln*dbln;

   temp2=temp2+k2;

   temp1=temp1/temp2;

   term=k1*temp1;

     Для того, щоб переконатися в правильному виводі другого результату, має сенс при налагодженні виводити разом із сумою й номер ітерації, на якому закінчилася робота циклу, і значення члена ряду, що виявилося менше межі точності:

    prіntf("%ld %10.7lf\n",n,term);
