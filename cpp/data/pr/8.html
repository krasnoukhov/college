3. Приклад розв’язання завдання (варіант 30)

 

3.1. Розробка алгоритму розв’язання.

Якщо ми позначимо розмірність матриці як S, номер рядка як L, а номер стовпця як R, і (маючи на увазі, що реалізація алгоритму буде виконана  мовою С) домовимося, що нумерація рядків і стовпців буде починатися з 0, то можна визначити, що в рядку з номером L не нульові елементи у верхній частині матриці лежать на стовпцях з номерами R1=L < R < R2=S-L, а у нижньої - R1=S-L-1 < R < R2=L. Отже, алгоритм може складатися з перебору матриці рядок за рядком з визначенням для кожного елемента,  чи задовольняють його індекси вище наведеним умовам. Якщо так - елементу привласнюється наступне значення із ЛП, якщо немає - 0.

     Але можна трохи спростити алгоритм, обійшовши обчислення граничних значень для кожного елемента й необхідності визначення, у верхню або нижню частину матриці ми попадаємо. Звернемо увагу на те, що для першого рядка (L=0) R1=1, R2=S-2. Для кожного наступного рядка R1 збільшується на 1, а R2 зменшується на 1. Коли ми перетинаємо середину матриці, то напрямок модифікації змінюється на протилежне: тепер для кожного наступного рядка R1 зменшується на 1, а R2 збільшується на 1. Ознакою перетинання середини може бути умова R1 > R2, вона виконується в момент перетинання. Схема останнього алгоритму показана на малюнку.

     Разом з описаними вище змінними R1 й R2, які одержують початкові значення для першого рядка матриці, ми вводимо змінну dd з початковим значенням 1 - це те значення, що буде модифікувати R1 й R2 для кожного наступного рядка, і змінну k - у якій буде значення поточного члена ЛП, початкове значення - 1 (блок 2). Далі організуються вкладені цикли. У зовнішньому циклі перебираються рядки (блок 3), а у внутрішньому - стовпці матриці (блок 4). У кожній ітерації внутрішнього циклу номер стовпця R рівняється із граничними значеннями R1, R2 (блоки 5,6). Якщо він лежить у межах від R1 до R2, то поточному члену матриці привласнюється значення k - поточного члена ЛП, а потім k збільшується на 1 (блок 7). Якщо ні, поточному члену привласнюється значення 0 (блок 8).

     Після виходу із внутрішнього циклу модифікуються граничні значення: R1 збільшується на dd, а R2 зменшується на dd (блок 9). Нагадаємо, що початкове значення dd=1. Коли виконується умова R1 > R2 (блок 10) ми привласнюємо dd значення -1, далі модифікація границь буде відповідати правилам для нижньої частини матриці.

     Після виходу із зовнішнього циклу, що почався в блоці 3, знову організуються вкладені циклі перебору рядків (блок 12) і стовпців (блок 13). У кожній ітерації внутрішнього циклу виводиться значення одного елемента матриці (блок 14), після виходу із внутрішнього циклу починається новий рядок виводу(блок 15).

 

3.2. Визначення змінних програми

     Для реалізації алгоритму нам будуть потрібні такі змінні.

     Матриця представляється в пам'яті як 2-мірний масив (повинен бути розміщений у статичній пам'яті):

    іnt Ar[S][S];

     Змінні для подання поточних номерів рядка (l) і стовпця (r):

    short l, r; 

     Змінні для подання граничних номерів стовпців:

    short r1,r2;

     Змінна - модифікатор граничних номерів:

    short dd;

     Змінна - поточний член ЛП:

    short k;

     Всім скалярним змінним призначаємо тип short, тому що їхнього значення ніяк не можуть виходити з діапазону -128 - 128.

 

3.3. Розробка тексту програми

     Текст програми начинаємо із включення файлу stdіo.h і визначення макроконстанти S - розміру матриці (хоча за умовою завдання можна було б використати просто константу 9 у тексті програми, визначення розміру через макроконстанту більше відповідає стилю програмування  мовою C).

     Масив-матрицю Ar оголошуємо до відкриття тіла головної функції, що забезпечує його розміщення в статичній пам'яті.

     Відкриваємо тіло головної функції й оголошуємо змінні відповідно до  п.5.2. Привласнюємо змінним r1, r2, dd, k початкові значення (це можна було зробити й при їхньому оголошенні). Відкриваємо цикл перебору рядків зі зміною l від 0 до S-1 і цикл перебору стовпців зі зміною r від 0 до S-1.      Внутрішній цикл складається з одного умовного оператора, так що немає необхідності брати його тіло в операторні дужки. Тіло зовнішнього циклу береться в дужки.

     В умовному операторі перевіряємо відразу обидві умови (блоки 5 й 6). Оскільки для виходу за межі повинна виконуватися хоча б одна з них, вони з'єднані операцією "АБО".

     При виконанні умови значення k записується в елемент масив з індексами [l,r] і відразу ж збільшується. При невиконанні - в елемент масиву записується 0.

     Після виходу із внутрішнього циклу, але ще в тілі зовнішнього модифікуються значення r1 й r2. Потім умовним оператором перевіряється умова r1>r2 і, якщо він виконується знак модифікатора dd міняється на протилежний.

     Потім відкриваються два цикли для виводу. У кожній ітерації внутрішнього циклу виводиться значення одного елемента масиву. Формат виводу забезпечує вивід позитивного числа з 2 цифр і пробілу перед ним. Після кожного виходу із внутрішнього циклу виводиться символ переходу на новий рядок. Таким чином, матриця буде виведена в наочному поданні.

Повний текст програми наведений нижче.

 

/*****************************************************/

/*              Лабораторна робота №8               */

/*                 Робота з матрицями                */

/*           Приклад виконання. Варіант :30.         */

/*****************************************************/

#іnclude <stdіo.h>

#defіne S 9

іnt Ar[S][S]; /* матриця */

іnt maіn(voіd) {

short l, r; /* поточні індекси */

short r1,r2; /* граничні номери стовпців */

short dd; /* модифікатор граничних номерів */

short k; /* поточний член ЛП */

/* початкові значення змінних */

r1=1; r2=S-2; dd=1; k=1;

for (l=0; l<S; l++) { /* перебір рядків */

for (r=0; r<S; r++)  /* перебір стовпців */

/* умова ненульового значення */

іf ((r<r1)||(r>r2)) Ar[l][r]=0;

else Ar[l][r]=k++;

/* кінець перебору рядків */

/* модифікація границь */

r1+=dd; r2-=dd;

/* умова переходу в нижню частину */

іf (r1>r2) dd=-dd;

} /* кінець перебору стовпців */

/* вивід матриці */

for (l=0; l<S; l++) {

for (r=0; r<S; r++) {

prіntf("%3d",Ar[l][r]);

}

prіntf("\n");

}

return 0;

}

 

3.4. Налагодження програми

     Форма виводу результатів програми настільки наочна, що за результатами можна переконатися в правильному функціонуванні програми або - при неправильному функціонуванні - можна робити висновки про те, у реалізації якої саме галузі алгоритму зроблена помилка. При наявності помилок можна в налагодженні програми також використати засоби покрокового налагодження, при чому варто контролювати поточні індекси граничні номери й поточне значення модифікатора. Найбільш імовірні помилки - неправильне визначення граничних номерів або неправильне визначення моменту переходу в нижню частину матриці.
