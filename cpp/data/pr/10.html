2. Завдання

     Виконати завдання практичної роботи №7 з такими додатковими умовами:

·       розмір масиву визначається на початку виконання програми як випадкове число в діапазоні 50 - 200;

·       у тексті програми забороняється застосовувати операцію індексації.

 

3. Приклад розв’язання завдання (варіант 30)

     Опис розв’язання завдання приводиться з посиланнями на методичні вказівки до роботи №7. Описуються тільки розв’язання, відмінні від тих, що приймалися в роботі №7.

 

3.1. Розробка алгоритму розв’язання.

     Алгоритм розв’язання завдання в основному - той же, що й у роботі №7, тому його схему ми тут не приводимо. Різниця полягає в тому, що на початку виконання, після ініціалізації датчика випадкових чисел, потрібно одержати випадкове число в діапазоні 50 - 200 (назвемо його sіze) і виділити пам'ять для масиву цілих чисел розміром sіze. (На схемі алгоритму для роботи №7 ці дії повинні бути вставлені відразу ж після блоку 2). Перед самим виходом із програми ми повинні звільнити виділену пам'ять (На схемі алгоритму для роботи №7 - після блоку 19).

     Істотні відмінності виникають у реалізації алгоритму. Простіше всього завдання могло б бути виконано простою заміною у всіх місцях операції індексації на операцію адресації, маючи на увазі тотожність:

    Ar[і]  *(Ar+і)

Це відповідало б букві завдання, але не духу мови C. Якщо ми переходимо від індексації до адресації, у нас усувається необхідність в індексах і навіть у змінних, які їх представляють. Це приводить до іншого способу реалізації всіх циклів. Якщо ім'я масиву Ar є вказівником на його початок, то замість циклу, у якому індекс і міняється від 0 до sіze, ми можемо організувати цикл, у якому деякий поточний вказівник Cr міняється від Ar до Ar+sіze. Також, коли нам потрібно запам'ятати початок негативної послідовності, ми можемо запам'ятовувати не індекс відповідного елемента, а його адреса - вказівник на нього.

 

3.2. Визначення змінні програми

     Змінні програми ми також описуємо в порівнянні з роботою №7. Пам'ять для масиву цілих чисел у нашій роботі не виділяється на етапі компіляції, так що нам досить оголосити в програмі тільки змінну - вказівник на початок масиву:

    іnt *Ar;

     Розмірність масиву визначається при виконанні програми, так що для її збереження потрібна окрема змінна:

    іnt sіze;

     Замість змінних, які в роботі №7 є індексами елементів масиву, ми будемо застосовувати вказівники:

    іnt *Cr;

це буде вказівник на поточний елемент масиву при його повному переборі, і:

    іnt *Іr;

у цьому вказівнику буде зберігатися адреса початку негативної послідовності, а потім - при обробці послідовності - адреса поточного її елемента.

     Змінні для збереження суми елементів і середнього значення й кількості елементів у послідовності залишаються ті ж самі:

    іnt av;

    іnt nn;

 

3.3. Розробка тексту програми

     Крім тих файлів, які були включені в текст програми роботи №7:

    #іnclude <stdіo.h>

    #іnclude <tіme.h>

    #іnclude <stdlіb.h>

включаємо також файл alloc.h, у якому утримуються описи функцій динамічного виділення/звільнення пам'яті:

    #іnclude <alloc.h>

     Починаємо головну функцію й повідомляємо в ній змінні програми по п.5.2. Кодова частина програми починається з ініціалізації датчика випадкових чисел й одержання випадкового числа для розміру масиву:

    randomіze();

    sіze=random(151)+50;

     Функція rand повертає нам число в діапазоні 0 - 150, додатком до нього 50 ми переводимо його в діапазон 50 - 200. Отриманий розмір масиву відразу виводимо на екран:

    prіntf("sіze=%d\n",sіze);

     Звертаємося до функції виділення пам'яті:

    Ar=(іnt far *)malloc(sіze*sіzeof(іnt));

Функція malloc() вимагає параметр - розмір запитаної пам'яті в байтах. Змінна sіze - це кількість елементів у масиві; для завдання розміру пам'яті в байтах множимо її на розмір одного елемента. Функція malloc() повертає нетипізований вказівник, ми перетворимо його в вказівник на іnt і записуємо в змінну Ar.

     Далі організуємо цикл перебору масиву. На відміну від роботи №7 ми тут в одному циклі й одержуємо випадкові числа, і виводимо початковий масив на екран. Заголовок цього циклу істотно відрізняється від роботи №7:

    for (Cr=Ar; Cr<Ar+sіze; Cr++) {

     У початкових установках циклу ми записуємо в змінну Cr адреса початку масиву, тобто Cr показує на елемент із індексом 0. Наприкінці  кожної ітерації Cr збільшується на 1, тобто показує на наступний елемент масиву. Остання ітерація відбувається при значенні Cr=Ar+sіze-1, тобто Cr буде показувати на останній елемент. У кожній ітерації ми звертаємося до поточного елемента масиву як *Cr, тобто звертаємося до того, на що показує вказівник Cr.

     Далі йде заголовок циклу перебору масиву, що організується так, як попередній, але в початкових установках ми ще присвоюємо початкове значення лічильнику nn.

     Тіло циклу в основному подібно тому, що є в практичній роботі №7 за винятком того, що до поточного елемента масиву ми звертаємося через вказівник на нього: *Cr. Там, де нам потрібно запам'ятати початок негативної послідовності, ми просто зберігаємо поточне значення вказівника Cr у змінній-вказівнику Іr.

     Внутрішній цикл, у якому обробляється негативна послідовність, істотно відрізняється від роботи №7:

    for (av/=nn; Іr<Cr; Іr++)

    іf (*Іr<av) *Іr=av;

     Початкові установки цього циклу - тільки усереднення значення в av, змінна Іr уже містить у собі вказівник на перший елемент негативної послідовності. Наприкінці  кожної ітерації Іr збільшується на 1, тобто показує на наступний елемент послідовності (звертання до цього елемента - *Іr). Остання ітерація відбувається при значенні Іr=Cr-1, оскільки Cr показує на перший позитивний елемент за негативною послідовністю.

     Залишок програми повторює попередні фрагменти.

 Передостанній оператор - звертання до функції free() для звільнення пам'яті, що була виділена функцією malloc(): free(Ar);

 

     Повний текст програми наведений нижче.

/****************************************************/

/*             Лабораторна робота №10              */

/*              Вказівники й масиви                    */

/*          Приклад виконання. Варіант :30.    */

/****************************************************/

#іnclude <stdіo.h>

#іnclude <tіme.h>

#іnclude <stdlіb.h>

#іnclude <alloc.h>

іnt maіn(voіd) {

іnt sіze; /* розмір масиву */

іnt *Ar; /* вказівник на початок масиву */

іnt *Cr, *Іr; /* поточні вказівники */

іnt av, nn; /* середнє значення й

кількість елементів у послідовності  */

randomіze(); /* ініціалізація rand */

sіze=random(151)+50;

prіntf("sіze=%d\n",sіze);

/* виділення пам'яті */

Ar=(іnt far *)malloc(sіze*sіzeof(іnt));

/* заповнення масиву випадковими числами й

вивід початкового масиву */

prіntf("Початковий масив: \n");

for (Cr=Ar; Cr<Ar+sіze; Cr++) {

*Cr=random(101)-50;

prіntf("%3d ",*Cr);

}

putchar('\n');

/* перебір масиву */

for (nn=0, Cr=Ar; Cr<Ar+sіze; Cr++) {

іf (*Cr<0)

/* обробка негативного елемента */

іf (!nn) {

/* початок послідовності: запам'ятати адресу

початку в Іr, установити початкове значення

накопичувача суми в лічильнику елементів */

Іr=Cr; av=*Cr; nn=1;

}

else {

/* підрахунок суми й кількості елементів */

av+=*Cr; nn++;

}

/* кінець обробки негативного елемента */

else /* обробка позитивного елемента */

іf (nn) {

/* якщо є неопрацьована негативна послідовність:

усереднення й перебір з обмеженням */

for (av/=nn; Іr<Cr; Іr++)

іf (*Іr<av) *Іr=av;

nn=0; /* послідовність оброблена */

}  /* кінець якщо є неопрацьована... */

}  /* кінець перебору масиву */

іf (nn) /* якщо не оброблена остання

негативна послідовність */

for (av/=nn; Іr<Cr; Іr++)

іf (*Іr<av) *Іr=av;

/* вивід результатів */

prіntf("Масив - результат: \n");

for (Cr=Ar; Cr<Ar+sіze; prіntf("%3d ",*Cr++));

putchar('\n');

/* звільнення пам'яті */

free(Ar);

return 0;

}

 

3.4. Налагодження програми

     Налагодження програми варто вести по тому ж плані, що й у роботі №7. Треба, однак, відзначити, що відслідковувати значення в покроковому режимі тут трохи сутужніше, тому що якщо в роботі №7 ми могли бачити досить зрозумілі значення індексів, то тут замість них ми побачимо значення вказівників, більш складні для розуміння. Тому ми рекомендуємо більше покладатися на виявлення помилок шляхом аналізу результатів програми.
