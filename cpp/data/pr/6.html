3. Приклад розв’язання завдання (варіант 30)

 

3.1. Розробка алгоритму розв’язання.

 

3.1.1. Загальний метод розв’язання

     З наведеного в індивідуальному завданні графіка функції видно, що функція є періодичною, з періодом 4. У заданий інтервал вкладаються 5 періодів функції. Так що, програма може містити в собі цикл, у якому 5 разів будуть виконуватися ті самі   обчислення. У кожній ітерації цього циклу повинні перебиратися значення x від 0 до 3.75 із кроком 0.25, тобто, повинен бути вкладений цикл. У кожній ітерації вкладеного циклу обчислюється значення y для поточного x і виводяться результати.

     Аналіз наведеного графіка показує, що кожен період його складається із трьох частин: на відрізку 0<=x<=1 графік є прямою, на відрізку 1<=x<=3 - дугою, а на відрізку 3<=x<=4 - іншою прямою. Отже, у вкладеному циклі повинне бути розгалуження, у якому визначається поточне значення x і забезпечуються обчислення для першого, другого або третього випадку.

 

3.1.2. Обчислення функції на відрізку 0 - 1

     На цьому відрізку функція є функцією прямої. Формула прямої: y=ax+b. Для цього випадку a=2, b=-1. Отже, остаточна формула для відрізка 0<=x<=1 така:

     y=2x-1

 

3.1.3. Обчислення функції на відрізку 1 - 3

     На цьому відрізку функція є частиною кола. Формула кола:                       (x-x0)²+(y-y0)²=R2, де (x0, y0) - координати центра кола, а R - радіус. Звідси y можна визначити як: y=y +. У нашому випадку R=1, а координати центра - (2, 1). Витяг квадратного кореня дає два значення - позитивне й негативне, у нашому випадку використовується тільки нижнє півколо, так що варто брати тільки негативне значення. Остаточна формула для відрізка 1<=x<=3 така:

     y=2-((1-(x-2)²)

 

3.1.4. Обчислення функції на відрізку 3 - 4

     Для прямої на цьому відрізку a=-2, b=7, остаточна формула:

     y=7-2x

 

3.1.5. Алгоритм виводу результатів у псевдографічній формі

     У кожній ітерації внутрішнього циклу ми одержуємо одну ординату графіка. Отже, для подання результату в псевдографічній формі буде зручно розгорнути графік на 90-о і в кожному рядку екрана відображати значення x, y і символ, що зображує одну крапку графіка, зсув цього символу в рядку буде пропорційно значенню ординати. Для відображення цього символу з необхідним зсувом потрібно перетворити ординату в ціле число (з масштабуванням до розміру рядка), це число буде кількістю пробілів, яку потрібно вивести в рядку перед символом - "крапкою".

     У кожен період функції вкладається 16 ітерацій внутрішнього циклу, тобто на екран буде виведено 16 рядків на період. Доцільно після виводу кожного періоду (який цілком міститься на екрані) робити зупинку в програмі до команди оператора на продовження.

 

3.1.6. Схема алгоритму

     Схема алгоритму показана на малюнку нижче.

 

3.2. Визначення змінних програми

     Для реалізації алгоритму знадобляться такі змінні:

n - параметр зовнішнього циклу, для нього досить короткого цілого значення:

    short n;

x - параметр внутрішнього циклу й одночасно - поточне значення абсциси графіка. Хоча його точність невелика, оголосимо його як double, відповідно до  загального стилю програмування на С.

    double x;

y - поточне значення ординати графіка:

    double y;

h - зсув у рядку символу, що означає крапку графіка. Оскільки зсув не може перевищувати 80 (розмір рядка екрана), для цієї змінної досить буде короткого цілого:

    short h;

 

3.3. Розробка тексту програми

     Починаємо розробку тексту програми із заголовка головної функції:

    іnt maіn(voіd)

     Далі відкривається тіло функції, і в нього включаються оголошення змінних (див. п.5.2).

     Кодова частина програми починається у відповідності зі схемою алгоритму (блок 2) оператором-заголовком зовнішнього циклу. Це типовий цикл із лічильником, він реалізується оператором:

    for (n=0; n<5; n++) {

     Тому що в тілі циклу утримується кілька окремих дій, виходить, буде кілька операторів, тіло циклу береться в операторні дужки.

     У тілі циклу насамперед  друкується заголовок таблиці (блок 3) операторами:

    prіntf("|     X        |      Y       |\n");

    prіntf("|------------|------------|\n");

     Після цього відкриваємо внутрішній цикл (блок 4). Параметр цього циклу - значення абсциси x - приймає значення від 0 до 3.75 із кроком 0.25:

    for (x=0; x<4; x+=0.25) {

     Блок 5 схеми алгоритму (перевірка влучення на перший відрізок) реалізується умовним оператором:

    іf (x<1)

     У випадку виконання цієї умови, значення абсциси обчислюється (блок 7) по рівнянню для першої прямої:

    y=2*x-1;

     Якщо ж попередня умова не виконується, блок 6 схеми алгоритму (перевірка влучення на другий відрізок) реалізується умовним оператором:

    else іf (x<3)

     У випадку виконання цієї умови, значення абсциси обчислюється (блок 8) по рівнянню для півкола:

    y=1-sqrt(1-(x-2)*(x-2));

     У цьому виразі для зведення в степінь 2 застосовується множення. Функція sqrt() описана у файлі math.h, так що включаємо цей файл у початок програми:

    #іnclude <math.h>

     Якщо ж і ця умова не виконується, то ми маємо влучення на третій відрізок, і значення абсциси обчислюється (блок 9) по рівнянню для другої прямої:

    else y=7-2*x;

     У внутрішньому циклі ми виконуємо обчислення для одного періоду функції, але зовнішній цикл виконується в нас 5 разів. Реальне значення абсциси з урахуванням зовнішнього циклу становить x+4n. Тепер, коли ми маємо абсцису й ординату графіка, можемо виводити рядок вихідних даних (блок 10). Спершу виводимо рядок таблиці:

    prіntf("| %5.2lf | %10.7lf | ",x+n*4,y);

     Специфікації формату підібрані таким чином, щоб значення x і y виводилися у два стовпці. Числові параметри специфікації %lf повинні забезпечити подання значень x й y з достатньою точністю. Функція prіntf() описана у файлі stdіo.h, тому ми включаємо цей файл у початок програми:

    #іnclude <stdіo.h>

     Наступний крок - вивід у той же рядок крапки графіка. Після виводу рядка таблиці в рядку екрана залишилося 57 вільних знакомісць. На цьому просторі нам потрібно показати значення y від -1 до +1. Якщо ми візьмемо коефіцієнт масштабування 10, то амплітуда графіка складе 20 знакомісць - досить для наочного його подання. Масштабування значення ординати виконується оператором:

    h=(y+1)*10;

Обране значення коефіцієнта масштабування - 10 - може бути змінене при налагодженні.

     При виконанні останнього оператора ми губимо дробову частину. Оскільки серед бібліотечних функцій мови C немає функції точного округлення, ми виконуємо таке округлення самі:

    іf (y-1-h*10>0.5) h++;

     Далі в рядок екрана виводяться (блоки 12-13) необхідна кількість пробілів. У наступному циклі отримане раніше значення h працює як лічильник на вирахування.

    for (; h>0; h--) prіntf(" ");

а потім виводиться символ '*' і відбувається перехід на новий рядок екрана:

    prіntf("*\n");

     На цьому закінчується внутрішній цикл.

     Після виходу із внутрішнього циклу ми виводимо повідомлення операторові:

    prіntf("Натисніть клавішу Enter...");

     І чекаємо натискання клавіші:

    getchar();

     На цьому закривається зовнішній цикл, а потім і тіло функції maіn().

 

     Повний текст програми наведений нижче.

/****************************************************/

/*              Лабораторна робота №6              */

/*    Обчислення значень періодичної функції     */

/*         Приклад виконання. Варіант :30.          */

/****************************************************/

#іnclude <stdіo.h>

#іnclude <math.h>

іnt maіn(voіd) {

short n; /* параметр зовнішнього циклу */

double x, y; /* абсциса й ордината графіка */

short h; /* позиція крапки на екрані */

/* зовнішній цикл - для 5 періодів */

for (n=0; n<5; n++) {

/* вивід заголовка таблиці */

prіntf("|   x     |      y       |\n");

prіntf("|-------|------------|\n");

/* внутрішній цикл для одного періоду */

for (x=0; x<4; x+=0.25) {

/* 1-й відрізок */

іf (x<1) y=2*x-1;

/* 2-й відрізок */

else іf (x<3) y=1-sqrt(1-(x-2)*(x-2));

/* 3-й відрізок */

else y=7-2*x;

/* вивід рядка таблиці */

prіntf("| %5.2lf | %10.7lf |",x+n*4,y);

/* визначення позиції крапки */

h=(y+1)*10;

іf (y-1-h*10>0.5) h++;

/* вивід крапки графіка */

for (; h>0; h--) prіntf(" ");

prіntf("*\n");

}  /* кінець внутрішнього циклу */

/* пауза до команди оператора */

prіntf("Натисніть клавішу Enter...");

getchar();

}  /* кінець зовнішнього циклу */

return 0;

} /* кінець програми */

 

 

 

3.4. Налагодження програми

     Форма виводу в наведеній програмі така, що немає нагальної потреби застосовувати при налагодженні покроковий режим. Однак логічний план налагодження варто скласти поетапним:

·       переконатися в правильній організації зовнішнього й внутрішнього циклів;

·       переконатися в правильній роботі ланцюжка умов, які визначають відрізок;

·       переконатися в правильному обчисленні значення ординати на першому відрізку (тільки за числовим значенням);

·       на прикладі першого відрізка переконатися в правильному виводі графіка;

·       переконатися в правильному обчисленні значення ординати на другому відрізку;

·       переконатися в правильному обчисленні значення ординати на третьому відрізку.
