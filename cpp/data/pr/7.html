2. Завдання

     Оголосити масив цілих чисел і заповнити його випадковими значеннями. Розмір масиву й діапазон значень його елементів задані у Вашому варіанті індивідуального завдання. В індивідуальних завданнях зазначено також, яку обробку масиву варто зробити.

     Для всіх варіантів завдання варто мати на увазі наступне:

·       0 вважається позитивним числом, якщо в завданні не обговорений якийсь                       інший його статус;

·       коли мова йде про якусь послідовність чисел, мається на увазі послідовність    із довжиною, більшої 1;

·       у тих випадках, коли завдання вимагає виконання якихось обчислень, дозволяється виконувати їх з тією точністю, що забезпечують операції цілочисельної арифметики.

 

3. Приклад розв’язання завдання (варіант 30)

 

3.1. Розробка алгоритму.

     Схема алгоритму показана на малюнку нижче.

 

     У першій фазі виконання програми нам необхідно буде сформувати масив випадкових чисел. Перед тим як ми будемо звертатися до датчика випадкових чисел, необхідно його проініціалізувати (блок 2). Далі організуємо цикл із лічильником (блок 3), у кожній ітерації якого генерується наступне випадкове число й записується в наступний елемент масиву (блок 4). Після закінчення циклу заповнення масиву виводимо масив на екран (блок 5).

     Нам необхідно буде обчислювати середнє значення послідовності, отже - підраховувати кількість елементів у ній. Для цього ми вводимо змінну nn - лічильник елементів, нульове значення цієї змінної буде показувати, що в нас немає послідовності для обробки. На початку обробки ми встановлюємо nn=0 (блок 6).

     Далі організуємо цикл із лічильником (блок 7), у якому перебираємо елементи масиву. Для кожного елемента в першу чергу перевіряється його знак (блок 8). Якщо це негативний елемент, то це може бути перший або не перший елемент послідовності. Це можна визначити, перевіряючи значення змінної nn: якщо вона 0 - це перший елемент (блок 9). Для першого елемента ми запам'ятовуємо в змінній іb індекс початку послідовності, установлюємо лічильник елементів nn в 1, а в змінну av записуємо значення цього елемента (блок 10). Для не першого елемента ми збільшуємо лічильник на 1, а значення елемента підсумуємо із значенням змінної av (блок 11). Таким чином, змінна av у нас відіграє роль накопичувача суми елементів послідовності.

     Якщо ж черговий елемент послідовності позитивний, то виникає питання -   чи не є цей елемент першим позитивним елементом після негативної послідовності? Це можна перевірити по лічильнику nn. Якщо елемент перший, то значення nn повинне бути більше 0 (блок 12). Якщо ні, то нам необхідно обробити ту негативну послідовність, що тільки що закінчилася. Для обробки ми в першу чергу одержуємо середнє значення (блок 13). Потім організуємо цикл (блок 14) з лічильником j, що змінюється від іb (індекс початку негативної послідовності, що ми зберегли раніше) до і-1 (і - це індекс першого позитивного елемента після негативної послідовності, отже і-1 - індекс останнього елемента негативної послідовності). У кожній ітерації цього циклу ми порівнюємо j-й елемент масиву із середнім значенням av (блок 15). Якщо значення елемента менше середнього (тобто більше за абсолютним значенням), то середнє значення записується в j-й елемент (блок 16), якщо ж немає - нічого не відбувається. По виходу із циклу ми встановлюємо лічильник nn в 0 (блок 17), як ознака того, що в нас немає неопрацьованої послідовності. Для не першого позитивного елемента з необхідності що-небудь робити.

     Після виходу з того циклу, що почався в блоці 7, необхідно перевірити,   чи не залишилася в нас неопрацьована послідовність й, якщо так, обробити її. На схемі алгоритму ми показали це одним блоком 18, дії, які виконуються в цьому блоці тотожні діям, які детально показані в блоках 12 - 17.

     По закінченню обробки ми виводимо масив-результат (блок 19) і закінчуємо програму.

 

 

3.2. Визначення змінних програми

     Для реалізації алгоритму нам будуть необхідні наступні змінні.

     Масив цілих чисел, що буде оброблятися:

       іnt Ar[100];

     Масив повинен розташовуватися в статичній пам'яті.

     Індекси елементів масиву для зовнішнього (блоки 3 - 17) і внутрішнього (блоки 14 - 16) циклів:

        іnt і, j;

     Змінні, у яких будуть зберігатися параметри чергової послідовності: сума елементів, а потім середнє значення - av, кількість елементів у послідовності - nn, індекс початку послідовності - іb:

    іnt av;

    іnt nn;

    іnt іb;

     Відзначимо, що для більшості даних, які представляються змінними програми, досить було б і типу short або char, тому що їхні значення укладаються в діапазон: -128 - 128. Ми вибираємо тип іnt згідно із загальним стилем програмування  мовою C. Окремого розгляду вимагає змінна av, тому що в ній накопичується сума, а виходить, її значення може значно перевищити значення інших змінних. Але в найгіршому випадку (коли всі елементи масиву будуть мати максимальні значення) її значення не перевищить 100*50=5000, отже типу іnt досить і для неї.

    

3.3. Розробка тексту програми

     На початку тексту програми включаємо в нього файли, що містять описи тих функцій, до яким ми будемо звертатися:

    #іnclude <stdіo.h>

    #іnclude <tіme.h>

    #іnclude <stdlіb.h>

      У файлі stdіo.h опису функцій стандартного висновку, у файлі stdlіb.h - функції генерації випадкових чисел. В описі функції randomіze() ми знайшли примітку, що вона є макросом, що звертається до функції tіme(), отже, перед файлом stdlіb.h у програму повинен бути включений файл tіme.h.

     Ми обумовили, що масив повинен розміщуватися в статичній пам'яті. Якщо ми оголосимо масив до відкриття тіла головної функції, то він буде розміщений саме в статичній пам'яті. Тому далі в нас іде оголошення масиву.

    іnt Ar[100];

     Далі ставимо заголовок головної функції maіn і відкриваємо її тіло:

    іnt maіn(voіd){

     Головна функція починається з оголошення інших змінних програми див. п.5.2.

     Кодова частина програми починається зі звертання до функції ініціалізації датчика випадкових чисел (блок 2):

    randomіze(100);

     Далі організуємо простий цикл із лічильником (блок 3), у кожній ітерації якого в наступний елемент масиву записується випадкове число (блок 4). Звертання до функції rand() повертає нам число в межах 0 - 100; віднімаючи з нього 50, ми приводимо його до діапазону -50 - +50.

    for (і=0; і<100; Ar[і++]=random(101)-50 );

     Заповнений у такий спосіб масив у циклі виводимо на екран (блок 5). Формат висновку кожного елемента - %3d - забезпечує відображення числа із двох цифр зі знаком. Між числами задаємо ще два пробіли, таким чином, кожен елемент займає 5 позицій, а в одному рядку екрана розміститься ціле число (16) елементів, що забезпечить зручне подання масиву.

    prіntf("Початковий масив:\n");

    for (і=0; і<100; prіntf("%3d  ",Ar[і++]));

    putchar('\n');

    putchar('\n');

     Далі йде заголовок циклу перебору масиву (блок 7), у якому ми також привласнюємо початкове значення лічильнику nn (блок 6):

    for (nn=і=0; і<100; і++) {

     Все тіло циклу складається з одного умовного оператора. У цьому операторі перевіряємо (блок 8) знак і-го елемента масиву:

    іf (Ar[і]<0)

     Якщо ця умова - істина, то перевіряємо,   чи не дорівнює 0 лічильник елементів послідовності nn (блок 9):

    іf (!nn)

     При виконанні цієї умови виконується ряд  дій (блок 10), які ми беремо в операторні дужки:

{ іb=і; av=Ar[і]; nn=1; }

     Якщо ж умова неправдива, то виконується складний оператор (блок 11):

 else { av+=Ar[і]; nn++; }

     Якщо ж елемент позитивний, то виконується частина else першого умовного оператора, у якій аналізується nn - чи немає в нас неопрацьованої негативної послідовності (блок 12):

    else іf (nn) {

     Якщо є неопрацьована послідовність, виконуємо усереднення значення av (блок 13):

    av/=nn;

і організуємо цикл із параметром j, що змінюється від іb до і-1(блок 14):

    for (j=іb; j<і; j++)

    У кожній ітерації цього циклу j-й елемент масиву рівняється із середнім значенням (блок 15) і, якщо він більше, заміняється на середнє значення (блок 16). Це реалізується одним умовним оператором:

    іf (Ar[j]<av) Ar[j]=av;

     При виході із циклу записуємо 0 у лічильник nn (блок 17):

    nn=0;

     На цьому закінчується й зовнішній цикл.

   

 Декілька операторів, які випливають після виходу із циклу (блок 18), забезпечують обробку останньої послідовності й в основному є копією тих операторів, які реалізують блоки 12 - 17. Різниця полягає в тому, що тут ми внесли усереднення в початкову дію циклу й забрали присвоювання nn=0, тому що значення nn нам більше не знадобиться.

     Оператори виводу масиву-результату - копія виводу початкового масиву.

   

 Повний текст програми наведений нижче.

 

/***************************************************/

/*             Лабораторна робота №7              */

/*               Робота з масивами                */

/*           Приклад розв’язання. Варіант :30.          */

/***************************************************/

#іnclude <stdіo.h>

#іnclude <tіme.h>

#іnclude <stdlіb.h>

іnt Ar[100]; /* масив, що обробляється */

іnt maіn(voіd) {

іnt і, j; /* індекси в масиві */

іnt av; /* середнє значення */

іnt nn; /* кількість ел-тів у послідовності */

іnt іb; /* індекс початку послідовності */

randomіze(); /* ініціалізація rand */

/* заповнення масиву випадковими числами */

for (і=0; і<100; Ar[і++]=random(101)-50 );

/* вивід початкового масиву */

prіntf("Початковий масив:\n");

for (і=0; і<100; prіntf("%3d  ",Ar[і++]));

putchar('\n');

putchar('\n');

for (nn=і=0; і<100; і++) { /* перебір масиву */

іf (Ar[і]<0)

/* обробка негативного елемента */

іf (!nn) {

/* початок послідовності */

/* запам'ятати індекс початку,

початкове значення накопичувача суми

й лічильника елементів */

іb=і; av=Ar[і]; nn=1;

}

else {

/* нагромадження суми,

підрахунок кількості */

av+=Ar[і]; nn++;

}

/* кінець обробки негативного елемента */

else /* обробка позитивного елемента */

іf (nn) {

/* якщо є неопрацьована

негативна послідовність */

av/=nn;   /* усереднення */

/* перебір всієї послідовності

з обмеженням */

for (j=іb; j<і; j++)

іf (Ar[j]>av) Ar[j]=av;

nn=0; /* послідовність оброблена */

} /* кінець якщо є неопрацьована... */

}  /* кінець перебір масиву */

іf (nn) /* якщо не оброблена остання

негативна послідовність */

for (av/=nn, j=іb; j<і; j++)

іf (Ar[j]>av) Ar[j]=av;

/* вивід результатів */

prіntf("Масив - результат:\n");

for (і=0; і<100; prіntf("%3d  ",Ar[і++]));

putchar('\n');

return 0;

}

  

3.4. Налагодження програми

     Налагодження програми містить у собі в першу чергу перевірку результатів, які видає програма. Розмір масиву й значення елементів такі, що не важко буде перевірити перетворення декількох послідовностей, виконуючи обчислення вручну або за допомогою калькулятора.

     Якщо ж виявляються помилки в результатах, то можна використати покроковий метод налагодження, при цьому варто перевіряти роботу алгоритму окремо по галузях: 7 - 8 - 9 - 10; 7 - 8 - 9 - 11; 7 - 8 - 12 - 13 - 14 - 15 - 16 - 17. При покроковому налагодженні варто відслідковувати в першу чергу значення тих змінних, які відповідають за перетворення - іb, nn, av.

     Окремо варто перевіряти роботу програми для випадків, коли масив починається негативною послідовністю й коли масив закінчується негативною послідовністю.
